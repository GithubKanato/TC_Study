# TC検定 勉強サイト 問題作成フォーマット

## 択一問題

`src/data/questions.js` の `questions` 配列に、次の形で追加します。

```js
{
  id: 79,
  category: 'カテゴリ名',
  prompt: '問題文。最も適切なものは、次のうちどれか。',
  options: [
    'アに表示される選択肢',
    'イに表示される選択肢',
    'ウに表示される選択肢',
    'エに表示される選択肢',
  ],
  answer: 0,
  explanation:
    '正解の理由と、他の選択肢が不適切な理由を短く説明します。',
},
```

### 入力ルール

- `id`: 既存の最後の番号の次を使います。
- `category`: ホームの分野別グラフに使われます。
- `prompt`: 例題形式に合わせて、原則 `次のうちどれか。` で終えます。
- `options`: 4つ書きます。画面では上から `ア`、`イ`、`ウ`、`エ` と表示されます。
- `answer`: 正解の番号です。`0` がア、`1` がイ、`2` がウ、`3` がエです。
- `explanation`: 回答後と復習問題に表示されます。

### 作成例

```js
{
  id: 79,
  category: '句読点',
  prompt: '読点を打つ目的として最も適切なものは、次のうちどれか。',
  options: [
    '文を飾るため',
    '係り受けや文の区切りを明確にするため',
    'すべての名詞の後に置くため',
    '句点の代わりに使うため',
  ],
  answer: 1,
  explanation:
    '読点は、文の構造や係り受けを読み取りやすくするために使います。打ちすぎる場合は文そのものを見直します。',
},
```

## 午後問題

`src/data/afternoonQuestions.js` の `afternoonCases` 配列に、次の形で追加します。

```js
{
  id: 'pm-new',
  title: '総合問題3 タイトル',
  category: '出題テーマ',
  overview:
    'この問題で確認する観点を書きます。例: 用語統一、範囲表現、認知特性を見直します。',
  sourceTitle: '原稿タイトル',
  rows: [
    { number: 1, content: '番号付き原稿の本文を入れます。' },
    { number: 2, content: '設問で指摘させたい不適切な表現を含めます。' },
    { number: 3, content: '必要に応じて複数行に分けます。' },
  ],
  questions: [
    {
      id: 1,
      category: '午後問題・カテゴリ名',
      prompt: '原稿のどこを指摘し、どのように直すかを問う設問文を書きます。',
      answerLabel: '番号と書き換え案',
      sampleAnswer:
        '番号2。「修正後の表現」のように書き換える。',
      explanation:
        'なぜその箇所が問題なのか、どの観点で直すのかを説明します。',
    },
  ],
},
```

### 午後問題の設問パターン

```js
{
  id: 1,
  category: '午後問題・用語統一',
  prompt: '同じ意味で異なる用語を使っている箇所を指摘し、用語を統一してください。',
  answerLabel: '番号と統一後の用語',
  sampleAnswer: '番号3と番号5。「利用者」に統一する。',
  explanation:
    '同じ概念に複数の用語を使うと、読み手が別の対象だと誤解する可能性があります。',
},
```

```js
{
  id: 2,
  category: '午後問題・範囲表現',
  prompt: '対象範囲があいまいな語句を1つ指摘し、確認すべき点を説明してください。',
  answerLabel: '語句と確認点',
  sampleAnswer: '番号4の「小学生以下」。小学生を含むか、無料対象との関係を確認する。',
  explanation:
    '「以上」「以下」「まで」などは境界を含むかどうかが問われやすい表現です。',
},
```

```js
{
  id: 3,
  category: '午後問題・認知特性',
  prompt: '短期記憶への負荷が高い箇所を指摘し、改善方法を説明してください。',
  answerLabel: '番号と改善方法',
  sampleAnswer: '番号6。長い列挙を箇条書きに分け、種類ごとに整理する。',
  explanation:
    '長い1文や長い列挙は、読み手が情報を保持しながら照合する負担を増やします。',
},
```
